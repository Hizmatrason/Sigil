# Sigil — План платформы лицензирования

Sigil — SaaS-платформа для выпуска, контроля и биллинга лицензий на программные продукты, в том числе работающие **оффлайн** на чужих серверах.

## Главные решения (зафиксированы)

- **Иерархия:** дерево `Company` произвольной глубины (через `parent_id`). Лицензия принадлежит любому узлу.
- **Backend:** ASP.NET Core 8 + EF Core + PostgreSQL — **всё self-hosted**, без managed RDS/Cloud SQL и т.п.
- **Frontend:** Vite + React + TypeScript + shadcn/ui.
- **Подпись лицензии:** Ed25519. (См. [03](03-licensing-protocol.md) — это **цифровая подпись**, не «шифрование приватным ключом», но эффект тот же: только владелец приватного ключа может выпустить валидную лицензию, клиент проверяет публичным.)
- **Хранение приватных ключей:** EncryptedFileSigner (master key из env) или HashiCorp Vault Transit — **никаких cloud KMS**.
- **Payload лицензии:** произвольный JSON (feature flags, числовые лимиты) + опциональный hardware fingerprint.
- **Оффлайн grace period:** 30 дней без сети сверх обычной валидации; общий `expires_at` — отдельно.
- **Единый API-эндпоинт для SDK:** все клиенты обращаются к `sigil.hizmatrason.tj` (поддомены на лицензию убраны).
- **Cloudflare** — единственный внешний сервис, и только в роли edge (TLS/WAF/DNS-зона), без API-интеграции. Всё остальное (PG, MinIO, Vault, Loki, Prometheus, SMTP, CI-runners, NuGet feed) — self-hosted.
- **Биллинг:** PDF-инвойсы + ручное/банковское признание оплаты. Stripe и другие SaaS-провайдеры платежей не используются.
