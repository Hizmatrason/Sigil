# Handover — Phase 0.3: EF Core + Npgsql + first migration

**Дата:** 2026-05-14
**Roadmap:** [Фаза 0 — Скелет](../10-roadmap.md), задачи «EF Core + Npgsql + первая миграция».

## Что сделано

### EF Core + Npgsql

- **`Sigil.Infrastructure`** теперь содержит:
  - `Microsoft.EntityFrameworkCore` 10.0.8
  - `Npgsql.EntityFrameworkCore.PostgreSQL` 10.0.1
  - `EFCore.NamingConventions` 10.0.0 (snake_case naming convention)
  - `Microsoft.EntityFrameworkCore.Design` 10.0.8 (runtime-only, для `dotnet ef`)

- **`SigilDbContext`** (`Sigil.Infrastructure/Data/SigilDbContext.cs`):
  - Пустой DbContext — без сущностей (они появятся в Phase 1).
  - `OnModelCreating` включает расширения PostgreSQL: `pgcrypto` и `ltree`.
  - Зарегистрирован в DI через `AddDbContext<>()` с `UseNpgsql()` + `UseSnakeCaseNamingConvention()`.

### Connection string

- `appsettings.json` получил секцию `ConnectionStrings:Default`:
  ```
  Host=localhost;Port=5432;Database=sigil;Username=sigil;Password=sigil_dev
  ```
  Совпадает с `docker-compose.dev.yml`.

### Health checks

- К существующим `/health` и `/health/live` добавлен Npgsql health-check для `/health/ready`:
  ```csharp
  builder.Services.AddHealthChecks()
      .AddNpgSql(connectionString!, name: "postgresql", tags: ["ready"]);
  ```
  Когда PostgreSQL доступен — `/health/ready` вернёт `200 OK`. Когда нет — `503 Service Unavailable`.

### Первая миграция

- `dotnet ef migrations add EnableExtensions` → миграция `20260514141520_EnableExtensions`.
- Генерирует SQL:
  ```sql
  CREATE EXTENSION IF NOT EXISTS ltree;
  CREATE EXTENSION IF NOT EXISTS pgcrypto;
  ```
- Расположение: `Sigil.Infrastructure/Data/Migrations/`.
- Файлы миграции вручную приведены к file-scoped namespace (наш `.editorconfig` требует `IDE0161`).

### docker-compose.dev.yml

- Создан `docker-compose.dev.yml` в корне репозитория:
  - **PostgreSQL 16** (alpine) → порт 5432, БД `sigil`, юзер `sigil`.
  - **Redis 7** (alpine) → порт 6379.
  - **MinIO** → порты 9000 (S3) и 9001 (Console UI).
  - **Mailpit** → порты 1025 (SMTP) и 8025 (Web UI).
  - Именованные volumes для persistence.
  - `healthcheck` на каждом сервисе.
- Запуск: `docker compose -f docker-compose.dev.yml up -d`.

### Тесты

- **`Sigil.Api.Tests`** — `HealthEndpointTests` обновлён:
  - Используется кастомная `SigilTestFactory` (наследник `WebApplicationFactory`), которая очищает `HealthCheckServiceOptions.Registrations` чтобы Npgsql health check не ломал тесты без доступной БД.
  - Все 3 теста (health, live, ready) проходят.
- **`Sigil.Infrastructure.Tests`** — добавлен `DatabaseMigrationTests`:
  - Интеграционный тест с Testcontainers PostgreSQL.
  - Запускает `postgres:16-alpine`, применяет миграцию, проверяет, что расширения `pgcrypto` и `ltree` установлены.
  - Требует запущенный Docker.
- Добавлен пакет `Testcontainers.PostgreSql` 4.6.0 в `Sigil.Infrastructure.Tests`.

## Принятые решения по ходу

1. **EF Core 10.0.8 vs Npgsql 10.0.1 (EF Core 10.0.4).** Npgsql.EntityFrameworkCore.PostgreSQL 10.0.1 зависит от EF Core 10.0.4. Мы явно ставим 10.0.8. Возникает MSB3277 warning о версии — не ломает работу, MSBuild выбирает 10.0.4 как primary. Обновим Npgsql до версии, совместимой с 10.0.8, когда она выйдет.

2. **`Microsoft.EntityFrameworkCore.Design` в Api.** EF tools (`dotnet ef`) требуют Design-пакет в startup-project. В Infrastructure он `PrivateAssets=all` — не транзитивен. Добавил в Api тоже.

3. **`SigilTestFactory` очищает `HealthCheckServiceOptions`.** Без этого `/health/live` и `/health/ready` возвращали `503` при отсутствии PostgreSQL. Для unit-тестов БД не нужна — поэтому очищаем регистрации.

4. **Миграции вручную приведены к file-scoped namespace.** `dotnet ef` генерирует block-scoped namespace, но наш `TreatWarningsAsErrors` + `IDE0161` ломают сборку. Каждый раз после `dotnet ef migrations add` нужно будет руками (или скриптом) фиксить namespace style.

5. **`docker-compose.dev.yml` вместо `docker-compose.yml`.** Файл предназначен только для локальной разработки. Production docker-compose появится в Phase 7.
