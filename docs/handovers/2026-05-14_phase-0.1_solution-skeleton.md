**Дата:** 2026-05-14
**Roadmap:** [Фаза 0 — Скелет](../10-roadmap.md), задача «Создать solution и проекты (Domain / Application / Infrastructure / Api / Client / SharedKernel + тесты)».

## Что сделано

Создана базовая структура решения на .NET 10 (`net10.0`).

```
Sigil/
├── global.json                       # пиннинг SDK 10.0.202
├── Directory.Build.props             # общие настройки (nullable, warnings-as-errors)
├── .editorconfig                     # стиль кода + игнор некоторых analyzer'ов
├── .gitignore                        # bin/obj/.env/keys/...
├── Sigil.slnx                        # solution (новый XML-формат .NET 10)
├── src/
│   ├── Sigil.SharedKernel/           # классы общего назначения
│   ├── Sigil.Domain/                 # → SharedKernel
│   ├── Sigil.Application/            # → Domain
│   ├── Sigil.Infrastructure/         # → Application
│   ├── Sigil.Api/                    # → Application + Infrastructure (web/Kestrel)
│   └── Sigil.Client/                 # → SharedKernel (NuGet SDK для клиентов)
└── tests/
    ├── Sigil.Domain.Tests/           # xUnit, → Domain
    ├── Sigil.Application.Tests/      # xUnit, → Application
    ├── Sigil.Infrastructure.Tests/   # xUnit, → Infrastructure
    ├── Sigil.Api.Tests/              # xUnit, → Api
    └── Sigil.Client.Tests/           # xUnit, → Client
```

Все проектные ссылки расставлены по Clean Architecture (см. [09-tech-stack.md](../09-tech-stack.md)). Класс `Sigil.Jobs` отдельным проектом **не выделен** — Hangfire-задачи поселятся в `Sigil.Infrastructure`, как и описано в плане.

В `Sigil.Api/Program.cs` пока минимальный код: один эндпоинт `GET /health → 200 { status: "ok" }`. Это временный заглушка, в Phase 0.2 заменю на полноценный setup с Serilog/OTel.

## Принятые решения по ходу

1. **.NET 10 вместо .NET 8.** В плане был указан .NET 8 LTS, но на машине пользователя установлены только .NET 9 (STS) и .NET 10 (LTS, последний). Перешёл на **.NET 10**. Документы это пока не отражают — см. «Что обновить в /docs» ниже.

2. **`Sigil.slnx` вместо `Sigil.sln`.** `dotnet new sln` в .NET 10 по умолчанию создаёт новый XML-based формат `.slnx`. Все `dotnet`-команды его прекрасно понимают. Если кому-то нужен legacy `.sln` — можно пересоздать через `dotnet sln migrate`, но смысла нет.

3. **Central Package Management (CPM) выключен.** Изначально я положил `Directory.Packages.props`, но шаблоны `dotnet new webapi` / `dotnet new xunit` подставляют `PackageReference Version="..."` напрямую, что конфликтует с CPM (`NU1008`). Включим обратно в Phase 0.2/0.3, когда руками задам версии и приведу `csproj`-ы к чистому виду без локальных `Version=`.

4. **Удалена boilerplate `WeatherForecast` из Api template** — он не проходил наш analyzer (CA1852).

5. **`OpenAPI` package удалён из `Sigil.Api.csproj`** — он сам подтянулся шаблоном, но в Phase 0.1 не нужен. Вернём осознанно в Phase 0.2.

6. **`Sigil.Api/Program.cs` объявляет `public sealed class Program`** в namespace `Sigil.Api` — нужно для `WebApplicationFactory<Program>` в `Sigil.Api.Tests` (потребуется позже для integration-тестов).

7. **`TreatWarningsAsErrors=true`** включён глобально. Это будет ловить нас на грязном коде сразу же.
